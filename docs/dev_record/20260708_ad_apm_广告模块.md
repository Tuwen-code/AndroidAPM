# 20260708 ad-apm 广告模块

## 需求概述

在现有 Android 多模块项目中新增独立广告模块，沉淀 Android 广告变现基础能力。模块需要支持开屏、激励视频、插屏、Banner、信息流等广告形态的统一接入，并为 AdMob、穿山甲等第三方广告平台预留 Adapter 扩展点。

本次实现不直接接入真实第三方 SDK，先提供稳定的核心抽象、调度策略和可测试的 Mock Adapter，避免引入外部 SDK 账号、仓库、隐私权限和构建不确定性。

## 方案设计

新增模块 `ad-apm`，命名空间为 `com.codex.adapm`。模块职责集中在广告商业化接入基础设施，不和现有内存监控模块耦合。

主要能力：

- 统一广告请求、加载、展示、回调事件模型。
- `AdNetworkAdapter` 抽象第三方平台 SDK 能力。
- `AdApmManager` 负责平台选择、预加载、展示、失败降级和事件分发。
- `AdCacheStore` 管理已加载广告缓存和过期判断。
- `AdFrequencyController` 处理插屏、开屏等广告展示频控。
- `RewardGuard` 基于 rewardId 做本地幂等保护，避免激励视频重复发奖。
- `MockAdNetworkAdapter` 用于 demo、单测和未接入真实 SDK 前的业务联调。

新增或修改文件：

- `settings.gradle.kts`
- `ad-apm/build.gradle.kts`
- `ad-apm/consumer-rules.pro`
- `ad-apm/src/main/AndroidManifest.xml`
- `ad-apm/src/main/java/com/codex/adapm/**`
- `ad-apm/src/test/java/com/codex/adapm/**`

## 风险与边界处理

真实 AdMob、穿山甲 SDK 需要 appId、广告位、隐私合规弹窗、权限、混淆和平台后台配置，本次不直接写死到核心库中。后续可新增独立平台模块，例如 `ad-apm-admob`、`ad-apm-pangle`，由它们实现 `AdNetworkAdapter`。

广告展示依赖 Activity 生命周期，核心模块接口保留 `Activity` 入参，但缓存、频控、奖励幂等等纯逻辑保持可 JVM 测试。模块不持久化 rewardId，当前幂等只覆盖进程内重复回调；线上补偿可通过业务层或后续 storage 扩展实现。

## 实现步骤

1. 创建开发记录和 `ad-apm` 模块配置。
2. 设计广告类型、状态、错误、展示结果、奖励订单等模型。
3. 实现 Adapter 抽象和管理器。
4. 实现缓存、频控、奖励幂等保护。
5. 添加 Mock Adapter 和核心 JVM 单测。
6. 运行 Gradle 测试和编译验证。

## 测试验证方案

重点验证：

- 激励广告多次触发 reward 回调时，同一 rewardId 只允许发奖一次。
- 频控命中时阻止广告展示，冷却时间后恢复展示。
- 首选平台加载失败时，管理器会切换到后备平台。
- 预加载成功后展示使用缓存广告。

执行命令：

- `.\gradlew.bat :ad-apm:testDebugUnitTest`
- `.\gradlew.bat :ad-apm:assembleDebug`
